# README 中文化设计

## 目标

将项目根目录的 `README.md` 尽量改为自然、专业的中文说明，同时保持项目事实、Markdown 结构和运行方式不变。

## 本地化范围

- 翻译标题、段落、列表、步骤说明和提示文字。
- 保留 `PaperPilot`、`Agent`、`RAG`、`React`、`TypeScript`、`Vite`、`FastAPI`、`Pydantic`、`DeepSeek`、`FastEmbed`、`BGE`、`JSON`、`DOCX`、`PostgreSQL` 等专业术语。
- 保留所有命令、代码块、环境变量、URL、端口、模型名称、文件路径及类名原样。
- 明确说明界面语言为中文，而生成的论文修改建议和回复审稿人内容为英文。
- 不新增未经代码或现有 README 支持的功能、性能指标或部署承诺。

## 文档结构

保持现有章节顺序，仅将章节名称中文化：项目简介、主要功能、技术架构、本地运行、环境配置、验证方式、产品流程和项目亮点。

## 验证

- 检查 Markdown 标题、列表和代码围栏是否完整。
- 对比修改前后的命令、环境变量和路径，确保字面内容不变。
- 检查技术术语和产品语言约定是否一致。
